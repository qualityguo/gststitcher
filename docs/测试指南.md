## 环境准备

### 依赖安装

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y \
    meson ninja-build \
    libglib2.0-dev \
    libgstreamer1.0-dev \
    libgstreamer-plugins-base1.0-dev \
    libjson-glib-dev \
    gstreamer1.0-tools \
    gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good \
    gstreamer1.0-plugins-bad
```

### 编译配置

```bash
cd gst-stitcher

# 配置构建（禁用CUDA，Phase 1仅支持CPU后端）
meson setup build -Dcuda=false

# 或重新配置
meson setup --reconfigure build
```

### 运行真实图片拼接测试

```bash
# 返回项目根目录
cd /home/ghq/gststitcher/gst-stitcher

# 设置插件路径（本地开发，不安装到系统）
export GST_PLUGIN_PATH=$(pwd)/build/gst

# 验证插件加载
gst-inspect-1.0 gststitcher

# 运行2路图片拼接测试
gst-launch-1.0 -e \
  gststitcher name=s \
    homography-file=tests/data/homographies_2img.json \
    border-width=10 \
  filesrc location=tests/data/ImageA.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA,width=640,height=480,framerate=1/1" \
    ! s.sink_0 \
  filesrc location=tests/data/ImageB.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA,width=640,height=480,framerate=1/1" \
    ! s.sink_1 \
  s. \
    ! videoconvert \
    ! pngenc \
    ! filesink location=tests/data/output_stitched.png

# 查看结果
echo "✓ 输出已保存到 tests/data/output_stitched.png"

# 验证输出文件
ls -lh tests/data/output_stitched.png
```

### 预期结果

- **输出尺寸**: 根据homography自动计算，通常在1000-1500像素宽度
- **拼接效果**: 两张图片应该平滑拼接，中间重叠区域有融合
- **文件大小**: 约500KB-2MB（取决于输出尺寸）

### 验证脚本

```bash
python3 << 'EOF'
from PIL import Image
import os
import sys

img_path = 'tests/data/output_stitched.png'

if os.path.exists(img_path):
    img = Image.open(img_path)
    print(f"✓ 输出图像尺寸: {img.size[0]}x{img.size[1]}")
    print(f"✓ 输出图像模式: {img.mode}")
    print(f"✓ 输出文件大小: {os.path.getsize(img_path) / 1024:.1f} KB")

    # 检查图像是否为空（全黑）
    pixels = list(img.getdata())
    avg_brightness = sum(sum(p[:3]) for p in pixels) / len(pixels)
    if avg_brightness > 10:
        print(f"✓ 图像内容正常 (平均亮度: {avg_brightness:.1f})")
    else:
        print(f"✗ 警告: 图像可能为全黑 (平均亮度: {avg_brightness:.1f})")
        sys.exit(1)
else:
    print("✗ 错误: 输出文件不存在!")
    sys.exit(1)
EOF
```

## 故障排查

### 问题1: 插件无法加载

**症状**: `gst-inspect-1.0 gststitcher` 报错 "No such element or plugin"

**解决方案**:
```bash
# 1. 检查环境变量
echo $GST_PLUGIN_PATH

# 2. 检查插件文件是否存在
ls -lh build/gst/libgststitcher.so

# 3. 检查符号表
readelf -s build/gst/libgststitcher.so | grep gst_plugin_desc

# 4. 强制重新扫描插件缓存
export GST_REGISTRY=/tmp/gst-registry-$(whoami).bin
gst-inspect-1.0 gststitcher
```

### 问题2: 单元测试失败

**症状**: `ninja -C build test` 失败

**解决方案**:
```bash
# 1. 清理重新编译
rm -rf build
meson setup build -Dcuda=false
ninja -C build
ninja -C build test

# 2. 单独运行失败的测试获取详细信息
./build/tests/test_warp_cpu
```

### 问题3: 输出图像为黑屏

**症状**: 拼接输出全黑

**解决方案**:
```bash
# 1. 检查输入格式
gst-launch-1.0 filesrc location=tests/data/ImageA.png ! pngdec ! videoconvert ! "video/x-raw,format=RGBA" ! fakesink

# 2. 启用调试日志
export GST_DEBUG=3,GST_STITCHER:4
gst-launch-1.0 -e ...

# 3. 检查homography配置
cat tests/data/homographies_2img.json | python3 -m json.tool

# 4. 验证图片文件
file tests/data/ImageA.png
file tests/data/ImageB.png
```

### 问题4: 运行时找不到libgststitcher.so

**症状**: `error while loading shared libraries: libgststitcher.so`

**解决方案**:
```bash
# 本地开发不需要设置LD_LIBRARY_PATH
# 只需设置GST_PLUGIN_PATH

export GST_PLUGIN_PATH=$(pwd)/build/gst

# 如果仍然有问题，检查插件路径
ls -la $(pwd)/build/gst/libgststitcher.so
```

### 问题5: 符号导出问题

**症状**: 插件编译成功但无法加载，日志显示符号未找到

**当前状态**: 已通过version-script修复，`gst_plugin_desc`应该正确导出

**验证**:
```bash
# 检查符号可见性
readelf -s build/gst/libgststitcher.so | grep gst_plugin_desc

# 应该看到GLOBAL符号:
# 11: 000000000000XXXX  NOTYPE  GLOBAL DEFAULT   XX gst_plugin_desc
```
