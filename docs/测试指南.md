## 环境准备

### 依赖安装

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y \
    meson ninja-build \
    libglib2.0-dev \
    libgstreamer1.0-dev \
    libgstreamer-plugins-base1.0-dev \
    libjson-glib-dev \
    gstreamer1.0-tools \
    gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good \
    gstreamer1.0-plugins-bad \
    libopencv-dev \
    pkg-config
```

### 编译配置

```bash
cd gst-stitcher

# 配置构建（禁用 CUDA，Phase 1 仅支持 CPU 后端）
meson setup build -Dcuda=false -Dcalibrate=true

# 或重新配置
meson setup --reconfigure build
```

## Map 文件模式测试

Map 文件模式是推荐的生产模式，性能最高。

### 1. 生成测试图片

```bash
cd gst-stitcher/tests/data

# 下载测试图片
bash download_test_images.sh

# 验证下载
ls -lh *.png
```

### 2. 生成 Homography 配置

```bash
# 使用标定工具生成 homography JSON
cd ../../

# 2 图标定
./build/calibrate/gst_stitcher_mapgen \
  -i tests/data/ImageA.png tests/data/ImageB.png \
  -o tests/data/homographies_2img.json \
  --format json
```

### 3. 生成 Map 文件

```bash
# 使用 homography JSON 生成 map 文件
./build/calibrate/gst_stitcher_mapgen \
  -i tests/data/ImageA.png tests/data/ImageB.png \
  -o tests/data/stitching.map \
  -f tests/data/homographies_2img.json \
  --format map
```

### 4. 查看 Map 文件信息

```bash
# 使用 map_info 工具查看 map 文件
./build/tools/map_info tests/data/stitching.map
```

**预期输出**：
```
Map File v2
============
Canvas: 1280x720
Images: 2
--------
Image 0: 640x480
  Offset: (0, 0)
  Crop: (0, 0, 0, 0)
--------
Image 1: 640x480
  Offset: (640, 0)
  Crop: (0, 0, 0, 0)
--------
```

### 5. 使用 Map 文件进行拼接

```bash
# 设置插件路径
export GST_PLUGIN_PATH=$(pwd)/build/gst

# Map 文件模式拼接
gst-launch-1.0 -e \
  gststitcher name=s \
    map-file=tests/data/stitching.map \
  filesrc location=tests/data/ImageA.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA" \
    ! s.sink_0 \
  filesrc location=tests/data/ImageB.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA" \
    ! s.sink_1 \
  s. \
    ! videoconvert \
    ! pngenc \
    ! filesink location=tests/data/output_map.png
```

### 6. 验证输出

```bash
# 检查输出文件
ls -lh tests/data/output_map.png

# 使用图像查看器查看
xdg-open tests/data/output_map.png 2>/dev/null || open tests/data/output_map.png
```

```json
        "reference": 0
      },
      "matrix": {
        "h00": 0.752823,
        "h01": -0.031433,
        "h02": 557.207642,
        "h10": -0.006882,
        "h11": 0.991056,
        "h12": 60.221836,
        "h20": -0.000021,
        "h21": -0.000018,
        "h22": 1.000000
      }
    }
  ]
}
```

**说明**: 以上 homography 矩阵为示例值，实际使用时需要通过标定工具计算。

```bash
  gststitcher name=s \

    border-width=10 \
  filesrc location=tests/data/ImageA.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA" \
    ! s.sink_0 \
  filesrc location=tests/data/ImageB.png \
    ! pngdec \
    ! videoconvert \
    ! "video/x-raw,format=RGBA" \
    ! s.sink_1 \
  s. \
    ! videoconvert \
    ! pngenc \
    ! filesink location=tests/data/output_json.png
```

## 故障排查

### 问题 1: 插件无法加载

**症状**: `gst-inspect-1.0 gststitcher` 报错 "No such element or plugin"

**解决方案**:
```bash
# 1. 检查环境变量
echo $GST_PLUGIN_PATH

# 2. 检查插件文件是否存在
ls -lh build/gst/libgststitcher.so

# 3. 检查符号表
readelf -s build/gst/libgststitcher.so | grep gst_plugin_desc

# 4. 强制重新扫描插件缓存
export GST_REGISTRY=/tmp/gst-registry-$(whoami).bin
gst-inspect-1.0 gststitcher
```

### 问题 2: 单元测试失败

**症状**: `ninja -C build test` 失败

**解决方案**:
```bash
# 1. 清理重新编译
rm -rf build
meson setup build -Dcuda=false
ninja -C build
ninja -C build test

# 2. 单独运行失败的测试获取详细信息
./build/tests/test_warp_cpu
```

### 问题 3: 输出图像为黑屏

**症状**: 拼接输出全黑

**解决方案**:
```bash
# 1. 检查输入格式
gst-launch-1.0 filesrc location=tests/data/ImageA.png ! pngdec ! videoconvert ! "video/x-raw,format=RGBA" ! fakesink

# 2. 启用调试日志
export GST_DEBUG=3,GST_STITCHER:4
gst-launch-1.0 -e ...

# 3. 检查 homography 配置
cat tests/data/homographies_2img.json | python3 -m json.tool

# 4. 检查 map 文件
./build/tools/map_info tests/data/stitching.map

# 5. 验证图片文件
file tests/data/ImageA.png
file tests/data/ImageB.png
```

### 问题 4: Map 文件加载失败

**症状**: "Failed to load map file" 错误

**解决方案**:
```bash
# 1. 检查 map 文件结构
ls -la stitching.map/
cat stitching.map/metadata.json

# 2. 使用 map_info 验证
./build/tools/map_info stitching.map

# 3. 检查文件权限
chmod -R 755 stitching.map/

# 4. 重新生成 map 文件
./build/calibrate/gst_stitcher_mapgen \
  -i img1.png img2.png \
  -o stitching.map \
  -f homographies.json \
  --format map
```

### 问题 5: 运行时找不到 libgststitcher.so

**症状**: `error while loading shared libraries: libgststitcher.so`

**解决方案**:
```bash
# 本地开发不需要设置 LD_LIBRARY_PATH
# 只需设置 GST_PLUGIN_PATH

export GST_PLUGIN_PATH=$(pwd)/build/gst

# 如果仍然有问题，检查插件路径
ls -la $(pwd)/build/gst/libgststitcher.so
```
