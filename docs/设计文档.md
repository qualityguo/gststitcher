## Context

参考 RidgeRun 的 cuda-stitcher 插件，从零实现一个功能类似的 GStreamer 拼接插件。

**目标**: 创建视频/图像拼接 GStreamer 插件，支持单应性矩阵变换 + 多路输入合成全景输出。

**用户确认的约束**:
- 目标平台: x86 + dGPU，支持 CPU 回退（通过参数切换）
- 无需适配 Jetson / NVMM
- MVP: 2路输入（已扩展支持 3+ 路）
- 需要同时开发单应性标定工具（已完成）

## 可行性评估：**可行**

核心算法（单应性透视变换、双线性插值、线性融合）是成熟算法，GStreamer GstAggregator 框架成熟，CPU/GPU 双路径实现可分阶段交付。

## 架构设计

### 三层架构 + 后端抽象

```
┌──────────────────────────────────────────┐
│        GStreamer Element Layer           │
│   gststitcher (GstAggregator 子类)       │
│   - Request Pads (sink_0..sink_N)        │
│   - 属性: homography-list, border-width  │
│   - Caps: video/x-raw, format=RGBA       │
├──────────────────────────────────────────┤
│        Stitching Logic Layer             │
│   - homography_config: JSON 解析         │
│   - canvas: 输出画布几何计算             │
│   - blender: 重叠区域融合协调            │
│   - warp_backend: 后端抽象接口           │
├──────────────┬───────────────────────────┤
│  CPU Backend │     CUDA Backend          │
│  (纯C实现)   │  warp_perspective.cu      │
│  bilinear插值│  blend_linear.cu          │
│  memcpy合成  │  cuda_utils.cu            │
└──────────────┴───────────────────────────┘
```

### 关键架构决策

1. **GstAggregator 基类** —— 内置多 sink pad 同步和 buffer 池管理
2. **后端抽象 (warp_backend)** —— CPU/GPU 通过函数指针或 vtable 切换，MVP 先实现 CPU 后端
3. **标准系统内存** —— 无需 NVMM，使用 `video/x-raw` 而非 `video/x-raw(memory:NVMM)`
4. **逆映射变换** —— 每个输出像素映射回源图像，避免空洞
5. **独立标定工具** —— 基于 OpenCV 的特征匹配 + 单应性计算

## 后端抽象接口

```c
// stitcher/warp_backend.h
typedef struct _StitcherBackend StitcherBackend;

struct _StitcherBackend {
    void* (*init)(int canvas_w, int canvas_h);
    void (*warp)(void* ctx,
                 const uint8_t* src, int src_w, int src_h, int src_stride,
                 uint8_t* dst, int dst_stride,
                 int dst_offset_x, int dst_offset_y,
                 int warp_w, int warp_h,
                 const float h_inv[3][3],
                 int crop_left, int crop_top);
    void (*blend)(void* ctx,
                  uint8_t* canvas, int canvas_stride,
                  int region_x, int region_y, int region_w, int region_h,
                  int border_width);
    void (*cleanup)(void* ctx);
};
```

**CPU 后端**: 纯 C 实现，使用标准内存操作
**CUDA 后端**: 内部使用 `cuMemAlloc`/`cuMemAllocHost`，通过 `cudaMemcpy` 传输

## Aggregate 主循环

```
1. 首次调用时:
   a. 解析 homography JSON → StitcherConfig
   b. 从各 sink pad caps 获取输入尺寸
   c. 计算画布几何 → CanvasInfo (输出尺寸 + 每图偏移)
   d. 初始化后端 (CPU 或 CUDA)
   e. 设置 src pad caps

2. 每帧:
   a. 分配输出 GstBuffer (画布尺寸, RGBA)
   b. map output buffer (GST_MAP_WRITE)
   c. 对每个 sink pad:
      - gst_aggregator_pad_pop_buffer() → GstBuffer
      - map input buffer (GST_MAP_READ)
      - 应用 crop (调整源指针和尺寸)
      - backend->warp(ctx, src, dst, offset, h_inv, crop)
      - unmap input
   d. 对每个重叠区域:
      - backend->blend(ctx, canvas, region, border_width)
   e. unmap output → gst_aggregator_finish_buffer()
```

## Canvas 几何计算算法

### 输入

- N 张输入图像尺寸: `(w_0, h_0), (w_1, h_1), ...`
- N-1 个 homography 矩阵: `H_0, H_1, ...`（每张 target 图到 reference 图的变换）
- 每张图的 crop 参数: `crop_left, crop_right, crop_top, crop_bottom`

### 算法步骤

```
1. 对每张图像计算其四角坐标（经过 crop 后）:
   四角 = [(0,0), (w-crop_right, 0), (0, h-crop_bottom), (w-crop_right, h-crop_bottom)]

2. 对每张图像的变换:
   - reference 图 (index=ref): 使用单位矩阵 I（不变换）
   - target 图: 使用给定的 H 矩阵

3. 将每张图四角坐标通过对应变换矩阵映射到输出空间:
   对于点 (x, y):
     p' = H * [x, y, 1]^T
     (x', y') = (p'_x / p'_z, p'_y / p'_z)

4. 收集所有图像映射后的四角坐标，计算整体 bounding box:
   min_x = min(所有映射后的 x')
   min_y = min(所有映射后的 y')
   max_x = max(所有映射后的 x')
   max_y = max(所有映射后的 y')

5. 画布尺寸:
   canvas_w = ceil(max_x - min_x)
   canvas_h = ceil(max_y - min_y)

6. 偏移量（确保所有坐标非负）:
   offset_x_i = -min_x  (全局偏移)
   offset_y_i = -min_y  (全局偏移)

   每张图在画布上的偏移:
     对 reference 图: offset_x_0 = -min_x, offset_y_0 = -min_y
     对 target 图: 经过 H 变换后的四角 + offset_x_0
```

### Warp 区域计算

每张图经变换后在画布上的实际像素范围（用于 warp 循环）：

```
warp_x_i = max(0, floor(变换后左上角 x + 全局偏移))
warp_y_i = max(0, floor(变换后左上角 y + 全局偏移))
warp_w_i = min(canvas_w - warp_x_i, ceil(变换后宽度))
warp_h_i = min(canvas_h - warp_y_i, ceil(变换后高度))
```

### 逆映射

warp 时对每个输出像素 `(ox, oy)` 需要映射回源图像坐标：

```
// 先减去全局偏移，得到变换空间坐标
(tx, ty) = (ox - offset_x, oy - offset_y)

// 逆变换回源图像坐标 (对于 target 图用 H_inv)
(src_x, src_y) = H_inv * (tx, ty)

// 再加上 crop 偏移
(src_x, src_y) = (src_x + crop_left, src_y + crop_top)
```

## 重叠区域检测与融合算法

### 重叠区域检测

两张图经 warp 后在画布上可能产生重叠区域：

```
对于图像 i 和图像 j (i < j):
1. 计算图像 i 在画布上的矩形区域: rect_i = (x_i, y_i, w_i, h_i)
2. 计算图像 j 在画布上的矩形区域: rect_j = (x_j, y_j, w_j, h_j)
3. 矩形求交:
   overlap_x = max(rect_i.x, rect_j.x)
   overlap_y = max(rect_i.y, rect_j.y)
   overlap_w = min(rect_i.x + rect_i.w, rect_j.x + rect_j.w) - overlap_x
   overlap_h = min(rect_i.y + rect_i.h, rect_j.y + rect_j.h) - overlap_y

4. 如果 overlap_w > 0 且 overlap_h > 0，则存在重叠区域
```

### 线性 Alpha 融合

在重叠区域内，使用线性加权平均：

```
对于重叠区域中的每个像素 (x, y):
  设 border_width 为融合过渡带宽度

  // 计算该像素在重叠区域中的相对位置
  // 以重叠区域的左边缘为 0，右边缘为 1
  t = (x - overlap_x) / overlap_w   // t ∈ [0, 1]

  // 限制融合过渡带范围
  if t < 0.5:
    alpha = max(0.0, t / (border_width / overlap_w))  // 左图渐出
  else:
    alpha = min(1.0, (t - 0.5) / (border_width / overlap_w) + 0.5)  // 右图渐入

  // 线性插值 (RGBA 四通道)
  output.R = (1 - alpha) * img_left.R + alpha * img_right.R
  output.G = (1 - alpha) * img_left.G + alpha * img_right.G
  output.B = (1 - alpha) * img_left.B + alpha * img_right.B
  output.A = 255  // 输出 alpha 始终为不透明
```

当 `border_width = 0` 时，不做融合，后 warp 的图像直接覆盖先 warp 的。

## GStreamer 时间戳同步与状态管理

### 时间戳同步策略

GstAggregator 基类内置了多 sink pad 的同步机制：

- **PTS 选择**: 取所有 sink pad 中**最早**到达的 PTS 作为输出帧的 PTS
- **时钟同步**: 由 GstAggregator 基类的 `gst_aggregator_wait()` 处理，等待所有 sink pad 数据就绪
- **丢帧处理**: 如果某个 sink pad 数据滞后，使用其最近一帧 buffer（`gst_aggregator_pad_peek_buffer`）
- **运行时长**: 从第一个 sink pad 的 running_time 计算

### 状态变更

```
NULL → READY:
  - 无特殊操作

READY → PAUSED:
  - 打开 homography-file（如果指定路径）
  - 不做解析，等待 caps 协商

PAUSED → PLAYING:
  - 首次 aggregate() 时执行初始化（延迟初始化）:
    a. 解析 homography JSON
    b. 获取各 pad caps，计算 canvas 几何
    c. 初始化后端
    d. 设置 src pad caps
  - 后续帧直接 warp + blend

PLAYING → PAUSED:
  - 保留后端状态，不释放资源

PAUSED → READY:
  - 释放后端资源 (backend->cleanup)
  - 清空配置

属性在 PLAYING 状态下:
  - homography-list/homography-file: 只读，设置后不可变
  - border-width: 可动态调整
  - backend: 不可切换（需回到 READY 重新初始化）
```

### Buffer 池

- 输出 buffer 使用 `gst_buffer_new_allocate()` 分配（GstAggregator 管理）
- 输出尺寸固定（首帧计算后不变），适合 buffer pool 复用
- 不手动创建 GstBufferPool，依赖 GstAggregator 基类管理

### 错误传播

```c
// GStreamer 错误系统
GST_ELEMENT_ERROR(element, LIBRARY, SETTINGS,
    ("Failed to parse homography JSON: %s", error->message),
    ("JSON parse error"));

GST_ELEMENT_ERROR(element, RESOURCE, NOT_FOUND,
    ("Homography file not found: %s", path),
    ("File access error"));

GST_ELEMENT_ERROR(element, STREAM, FORMAT,
    ("Input caps mismatch: all sinks must have same framerate"),
    ("Caps negotiation failed"));
```

## 错误处理与边界情况

### JSON 解析错误

| 错误场景 | 处理方式 |
|---------|---------|
| JSON 语法错误 | GST_ELEMENT_ERROR → 停止 pipeline |
| homographies 数组为空 | GST_ELEMENT_ERROR |
| target == reference | GST_ELEMENT_ERROR（日志提示具体哪个条目）|
| 重复 target index | GST_ELEMENT_ERROR |
| index 超出范围 [0, N-1] | GST_ELEMENT_ERROR |
| 矩阵元素缺失 | GST_ELEMENT_ERROR |

### Warp 边界情况

| 场景 | 处理方式 |
|------|---------|
| 源坐标超出源图像范围 | 输出透明像素 (R=0, G=0, B=0, A=0) |
| 退化矩阵 (det ≈ 0) | 初始化时检测，GST_ELEMENT_ERROR |
| 两张图无重叠 | 正常 warp，不执行 blend |
| 两张图完全重叠 | blend 覆盖整个画面 |
| 不同分辨率输入 | 允许，各自 warp 到各自区域 |
| crop 后尺寸 ≤ 0 | GST_ELEMENT_ERROR |

### 运行时错误

| 场景 | 处理方式 |
|------|---------|
| sink pad 数量与 homography 不匹配 | 初始化时检测并报错 |
| 某个 sink pad 无数据 | 跳过该帧，用上一帧或黑色填充 |
| 后端初始化失败 (CUDA OOM) | 回退到 CPU 后端 + GST_WARNING |

## Alpha 通道处理

### 输入 Alpha

- 输入 Caps 要求 RGBA，alpha 通道在 warp 时**透传**
- 源图像范围外的 warp 像素: alpha 设为 0（完全透明）
- 源图像范围内的 warp 像素: alpha 保持原值

### 融合时 Alpha

- 融合仅对 RGB 通道做线性插值
- 融合区域的输出 alpha 始终设为 **255**（不透明）
- 理由：拼接输出的目的是生成完整画面，半透明无意义

### 输出 Alpha

- 所有非重叠区域: alpha = 255（不透明）
- 画布中无图像覆盖的区域: alpha = 0（透明），便于后续叠加

## 标定工具设计

### 使用方式

```bash

  s. ! jpegenc ! filesink location=output.jpg
```

### 标定流程

```
1. 加载两张输入图像
2. 转灰度
3. ORB 特征检测 → keypoint + descriptor
4. BFMatcher 特征匹配
5. Lowe's ratio test 过滤匹配
6. findHomography(RANSAC) → 3x3 单应性矩阵
7. 验证: 内点比例 > 阈值
8. 输出 JSON
```
